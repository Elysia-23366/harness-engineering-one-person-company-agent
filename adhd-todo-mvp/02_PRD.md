# PRD · FocusCut MVP
> 文件编号：02 | 作者：R01 AI 产品经理 | 版本：v1.0 | 日期：2025-01

---

## 1. 产品定位

**一句话**：AI 替代 ADHD 用户的"任务规划"认知步骤，让他们永远只看到下一个可执行动作。

**北极星指标**：用户首次任务启动时间（从输入任务 → 开始执行第一个子任务）≤ 8 分钟

---

## 2. 用户故事（核心 3 条，MVP 只做这 3 条）

| # | 用户故事 | 验收标准 |
|---|----------|----------|
| US-01 | 作为 ADHD 用户，我想输入一个模糊任务，AI 在 5 秒内给我拆成可执行步骤 | 拆解响应 ≤ 5s；子任务 ≤ 5min/条；每条 ≤ 15 字；动词开头 |
| US-02 | 作为用户，我想每次只看到"当前这一件事"，不被列表淹没 | 主界面永远只展示 1 条当前子任务；其余折叠 |
| US-03 | 作为用户，我想在 AI 拆解结果不满意时重新生成或手动编辑 | "重拆"按钮一键触发；子任务可点击编辑 |

---

## 3. 功能清单（MoSCoW）

### Must Have（MVP 必须有）
- **M1** 任务输入：文字输入框 + 语音转文字（调用系统 API）
- **M2** AI 拆解引擎：调用 OpenAI GPT-4o，按提示词策略输出结构化子任务
- **M3** 单任务聚焦视图：主界面只显示当前子任务 1 条
- **M4** 子任务完成交互：滑动/点击完成，动画反馈，自动展示下一条
- **M5** 重新拆解：一键重新生成全部子任务
- **M6** 手动编辑子任务：点击任意子任务可编辑文字
- **M7** 精力状态采集：打开 App 时 1 个三档滑块（可跳过）
- **M8** 任务列表：查看所有未完成任务（入口在主界面次要位置）
- **M9** 本地数据持久化：任务和子任务存本地（SQLite / AsyncStorage）

### Should Have（时间允许加）
- **S1** 完成庆祝动画（多巴胺反馈）
- **S2** 今日完成统计（"今天搞定了 X 件事"）

### Won't Have（MVP 明确不做）
- 日历集成、提醒推送、团队协作、习惯追踪、番茄钟、标签/项目系统

## 4. AI 拆解引擎规格（核心差异化，R03 必读）

### 4.1 输入处理
- 接受自然语言，包括模糊输入（"搞那个 PPT"）
- 若输入 < 5 字或极度模糊，AI 触发 1 次追问（最多 1 次，不能更多）
- 支持中英文混合

### 4.2 输出规格（硬性约束）
```
每个子任务必须满足：
- 预估时长 ≤ 5 分钟
- 文字长度 ≤ 15 字（中文）
- 以动词开头（打开、写、找、发、确认…）
- 无歧义，可直接执行
- 子任务数量：3–7 条（不足 3 条说明任务太小，不需拆；超过 7 条继续递归）
```

### 4.3 精力状态适配
| 精力档位 | AI 策略 |
|----------|---------|
| 低 | 子任务时长压缩至 ≤ 2 分钟；优先排"最简单"的步骤 |
| 中 | 默认策略 |
| 高 | 允许子任务时长至 5 分钟；按逻辑顺序排列 |

### 4.4 System Prompt（锁定版，R03 直接用）

```
你是一个专为注意力困难用户设计的任务拆解助手。

规则（必须严格遵守）：
1. 将用户输入的任务拆解为 3-7 个子任务
2. 每个子任务：≤15个中文字，以动词开头，预估完成时间≤5分钟
3. 子任务必须具体可执行，不能有歧义
4. 如果某个子任务仍然超过5分钟，继续拆解它（最多递归3层）
5. 精力状态：{{energy_level}}（低/中/高），据此调整子任务顺序和时长
6. 输出格式为 JSON，不要输出任何其他内容

输出格式：
{
  "tasks": [
    {"id": 1, "action": "打开 Word 新建文档", "estimated_minutes": 1},
    {"id": 2, "action": "写第一段开头句", "estimated_minutes": 3}
  ]
}

如果用户输入过于模糊（少于5个字或无法判断任务类型），输出：
{
  "need_clarification": true,
  "question": "你想做的是什么类型的任务？（一句话描述）"
}
```

### 4.5 错误处理
| 场景 | 处理方式 |
|------|----------|
| API 超时（>8s） | 展示 loading + "AI 在思考，稍等一下"；超过 15s 提示重试 |
| API 返回格式错误 | 前端 fallback：展示原始任务 + "AI 开小差了，要重试吗？" |
| 网络断开 | 提示"当前离线，可手动添加子任务" |

## 6. 关键交互规则（给 R02 的设计约束）

1. **认知负荷第一原则**：任何界面的信息元素 ≤ 3 个
2. **主操作永远只有 1 个**：当前界面最显眼的按钮只能有一个
3. **不用红色**：ADHD 用户对压力色敏感，错误提示用橙色+文字，不用红色警告图标
4. **动画时长**：完成动画 ≤ 600ms，不能让用户等
5. **字体大小**：主任务文字 ≥ 20pt，保证低精力状态下也能轻松阅读
6. **输入框自动聚焦**：进入输入页面立即弹出键盘，减少点击步骤

---

## 7. 技术约束（给 R03 的边界）

- 平台：iOS + Android（React Native 或 Flutter，R03 自选）
- AI API：OpenAI GPT-4o（需申请 API Key，MVP 阶段费用由产品侧承担）
- 本地存储：优先本地持久化，不强依赖后端（MVP 无用户账号系统）
- 最低支持系统：iOS 15+ / Android 10+
- 语音输入：调用系统原生 STT，不自建

## 8. 北极星指标 + 指标体系

### 北极星指标
**任务启动时间**：从用户输入任务 → 点击"完成"第一个子任务，目标 ≤ 8 分钟

### 一级指标（产品健康度）
| 指标 | 定义 | MVP 目标 |
|------|------|----------|
| D1 留存率 | 次日打开 App 的用户比例 | ≥ 40% |
| 任务完成率 | 创建任务后完成 ≥ 1 个子任务的比例 | ≥ 70% |
| AI 拆解接受率 | 用户未点"重拆"直接开始执行的比例 | ≥ 60% |

### 二级指标（诊断用）
| 指标 | 定义 | 用途 |
|------|------|------|
| 平均子任务完成数/session | 每次使用完成几个子任务 | 判断聚焦效果 |
| 重拆率 | 点击"重新拆解"的比例 | 诊断 AI 质量 |
| 手动编辑率 | 编辑子任务的比例 | 诊断拆解粒度 |
| 精力状态分布 | 低/中/高各占比 | 了解用户使用场景 |
| 崩溃率 | Crash / session | 质量底线，目标 < 0.1% |
