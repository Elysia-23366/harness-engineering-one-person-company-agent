## 3. 功能范围

### ✅ IN SCOPE（必须有）

| # | 功能 | 说明 |
|---|------|------|
| F1 | 对话输入框 | 支持回车发送 + 发送按钮，输入为空不可发送 |
| F2 | 消息气泡展示 | 用户消息右对齐，客服消息左对齐，带头像/角色标识 |
| F3 | 模拟回复逻辑 | 关键词匹配 → 预设回答；无匹配 → 兜底话术 |
| F4 | 打字机效果 | 客服回复逐字显示，模拟真实 AI 输出感 |
| F5 | 会话历史 | 当次页面内保留完整对话记录，可滚动查看 |
| F6 | 加载状态 | 发送后显示"客服正在输入…"，再显示回复 |

### ❌ OUT OF SCOPE（明确不做）

- 用户登录 / 身份验证
- 对话持久化存储（数据库）
- 多客服坐席 / 转人工
- 移动端适配（桌面优先）
- 多语言切换
- 真实 OpenAI API 接入（预留接口，不强制）

## 4. 模拟回复规则（R03 直接实现）

```
关键词匹配表（优先级从上到下）：

"退款" / "退货"     → "您好，退款申请一般在3个工作日内处理，请提供订单号，我来帮您查询。"
"发货" / "物流" / "快递" → "您的订单已发货，物流信息会在24小时内更新，请稍候查看。"
"优惠" / "折扣" / "券"  → "目前平台有满减活动，可在首页查看最新优惠信息。"
"人工" / "转人工"    → "正在为您转接人工客服，预计等待时间2分钟，请稍候。"
"你好" / "您好" / "hi" / "hello" → "您好！我是智能客服小助手，请问有什么可以帮您？"
"谢谢" / "感谢"     → "不客气，如有其他问题随时告诉我！"
兜底（无匹配）        → "感谢您的提问，我已记录您的问题，稍后会有专员跟进处理。请问还有其他需要帮助的吗？"
```

## 6. 交互规范

| 交互 | 行为 |
|------|------|
| 回车键 | 发送消息（Shift+Enter 换行） |
| 发送按钮 | 点击发送，发送中置灰 |
| 空消息 | 不触发发送，输入框轻抖动提示 |
| 发送后 | 输入框清空，焦点保持在输入框 |
| 新消息 | 自动滚动到最新消息 |
| 客服回复延迟 | 600ms~1200ms 随机，模拟真实感 |

## 9. 北极星指标 + 指标体系

> ⚠️ Demo 阶段无真实用户数据，指标用于**演示验收**和**后续迭代参考**

**北极星指标**：演示成功率（Demo 过程中无报错、对话流畅完成的场次占比）→ 目标 100%

| 层级 | 指标 | 目标值 | 说明 |
|------|------|--------|------|
| 北极星 | 演示成功率 | 100% | 无 JS 报错，对话可完整进行 |
| 一级 | 首次回复时间 | ≤ 1.5s | 从发送到看到"正在输入"的时间 |
| 一级 | 关键词命中率 | ≥ 80% | 预设关键词能正确触发对应回复 |
| 二级 | 页面加载时间 | ≤ 1s | 本地打开，无网络依赖 |
| 二级 | 输入框可用性 | 100% | 发送后焦点正确回归输入框 |
